Props en React

En React, los props son como argumentos que le pasas a una función, pero en el contexto de los componentes. Son objetos que contienen información y se utilizan para enviar datos desde un componente padre a un componente hijo.

¿Para qué sirven los props?

  1. Comunicación entre componentes: Son la principal forma en que los componentes padres pueden "hablar" con sus componentes hijos, enviándoles la información que necesitan para renderizarse o funcionar correctamente.

  2. Personalización y reutilización de componentes: Los props permiten que un mismo componente hijo pueda mostrar información diferente dependiendo de los props que le envíe su componente padre. Esto hace que los componentes sean más dinámicos y reutilizables en diferentes partes de la aplicación.
  3. Control del comportamiento de los componentes hijos: Los componentes padres pueden usar props para controlar cómo se comportan sus componentes hijos. Por ejemplo, podrían pasar un prop booleano para indicar si un botón debe estar habilitado o deshabilitado.

En resumen:
Los props son fundamentales en React porque permiten la transferencia de datos unidireccional desde los componentes padres hacia los componentes hijos, lo que facilita la creación de interfaces de usuario dinámicas, reutilizables y bien organizadas. Piensa en ellos como los parámetros que configuran cómo se comporta y qué muestra un componente hijo.

Componente Padre



                import React from 'react';
                import Saludo from './Saludo'; // Asegúrate de que el archivo Saludo.js exista en la misma carpeta o en la ubicación correcta
                
                function App() {
                  const nombre = "María";
                
                  return (
                    <div>
                      <Saludo nombre={nombre} />
                      <Saludo nombre="Pedro" /> {/* Puedes enviar diferentes props al mismo componente */}
                    </div >
                  );
                }
                
                export default App;

Componente Hijo


                import React from 'react';
                
                function Saludo(props) {
                  return <h1>Hola, {props.nombre}!</h1>;
                }
                
                export default Saludo;

Explicacion

  • Paso de Props: En el componente App, estamos pasando el prop nombre al componente Saludo utilizando la sintaxis de atributos HTML:
  • Recepción de Props: En el componente Saludo, la función recibe un objeto llamado props como argumento. Podemos acceder al valor del prop nombre utilizando props.nombre.
  • Renderizado: El componente Saludo utiliza el valor del prop nombre para renderizar el saludo dinámicamente.
  • Reutilización: Puedes reutilizar el componente Saludo con diferentes valores de props, como se muestra en el segundo uso del componente en App.
  • Unidireccionalidad: Los props fluyen de arriba hacia abajo, es decir, del componente padre al hijo. No puedes modificar los props dentro del componente hijo.
