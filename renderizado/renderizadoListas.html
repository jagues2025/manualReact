Renderizado de listas

En este ejemplo, estamos iterando sobre el array frutas con map() y generando un <li> por cada fruta.

Nota: El key={index} ayuda a React a identificar cada elemento de manera √∫nica, pero si tienes un id √∫nico en los datos, es mejor usarlo en lugar del √≠ndice.


            import React from "react";

                function ListaFrutas() {
                const frutas = ["Manzana", "Banana", "Naranja", "Uva"];

                return (
                    <ul>
                    {frutas.map((fruta, index) => (
                        <li key={index}>{fruta}</li>
                    ))}
                    </ul>
                );
                }

                export default ListaFrutas;

Ejemplo con objetos y una clave √∫nica

Si los elementos vienen como objetos con una propiedad id, se recomienda usar ese id como clave (key).

Ventaja de usar id como key: React puede optimizar la actualizaci√≥n de la lista sin re-renderizar innecesariamente los elementos.


            import React from "react";

                function ListaUsuarios() {
                const usuarios = [
                    { id: 1, nombre: "Ana" },
                    { id: 2, nombre: "Juan" },
                    { id: 3, nombre: "Carlos" }
                ];

                return (
                    <ul>
                    {usuarios.map((usuario) => (
                        <li key={usuario.id}>{usuario.nombre}</li>
                    ))}
                    </ul>
                );
                }

                export default ListaUsuarios;
