Uso del Fragment en React

En React, un Fragment (React.Fragment o su versión corta <>) permite agrupar múltiples elementos hijos sin añadir un nodo extra al DOM. Esto es útil cuando no quieres que el contenedor afecte el diseño o el rendimiento de tu aplicación

Un Fragment puede tener atributos como cualquier otro elemento de React. Un uso común es con la propiedad key para asignar una clave a los fragmentos creados dinámicamente

Por que usar Fragment

Por defecto, JSX necesita que el contenido dentro de un componente esté envuelto en un único contenedor. Si no necesitas un contenedor como un <div> (que crea un nodo en el DOM), puedes usar un Fragment para evitarlo.


                function App() {
                    return (
                      <Fragment>
                        <h1>Título principal</h1>
                        <p>Este es un párrafo adicional.</p>
                      <Fragment/>
                    );
                  }
                  
                  export default App;

Esto es lo mismo que el anterior codigo


                function App() {
                    return (
                      <> // Este es la abreviatura del Fragment
                        <h1>Título principal</h1>
                        <p>Este es un párrafo adicional.</p>
                      </>
                    );
                  }
                  
                  export default App;
